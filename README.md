#usage
To use akgwas, you need to prepare the following files:
Required:
    1. Raw sequencing files or assembled genomes​ (see example). Prepare an fof file based on the raw files (one line per sample; one sample can have multiple fasta files, both fasta and fasta.gz are supported):
    Example of fof.txt：
       sample_name : /path/to/sequencing/pseudo88_1.fasta.gz ; /path/to/sequencing/pseudo88_2.fasta.gz ; /path/to/sequencing/pseudo88_n.fasta.gz
    2. Phenotype file, which must correspond to the samples in the fof file in both number and order. 
       Each phenotype should be saved in a separate txt file containing a single column of phenotype values, with NAused to fill in missing data.
    3. config.yaml file, which stores all parameters. Use the command akgwas --init-configto generate the initial config.yaml file, and modify it as needed.
    4. Prepare bwa index​ according to the reference genome to be used. Then create a ref_list.txtfile, with each line containing the path to a prepared bwa index .sa record (e.g., /home/suz/sdc/Reference_index/Arabi/pseudo10001.fasta.gz.sa).
Optional:
    5. Pre‑generated kinship and PCA files, whose sample number and order must match those in the fof file (e.g., generated from SNP genotyping data).
       If not provided, kinship and PCA will be automatically generated by random sampling of the k‑mer matrix, but the results may vary between runs, thus affecting GWAS structure.
       You can back up the randomly generated kinship and PCA files (in config.yaml: QK_path: "QK") to ensure reproducibility.
